## 关键技术决策

### 1. 暗色配色方案：双色板（不用 HSL 运行时推导）

`CategoryColors.colorFor(key)` 改为 `@Composable`，内部根据 `isSystemInDarkTheme()` 在 lightPalette / darkPalette 之间切。darkPalette 用 Material 200 调色板的 light variant（如 blue `#1976D2` → `#90CAF9`），保证暗色背景上对比度 ≥ 4.5:1。

**为什么不用 HSL 自动提亮**：手写 HSL 转换 + lightness 抬升 + saturation 校正在 Material 调色板上效果不如直接挑 M3 推荐值；而且只有 8 个 colorKey，硬编码更可读。

**调用方安全检查**：3 个调用点（`CategoryPicker.kt:49` / `TimelineBlock.kt:31` / `StatsScreen.kt:69`）全部在 `@Composable` 上下文，可安全切换。

### 2. Lint 修复方案：core library desugaring

`./gradlew lint` 第一次报 94 个 `NewApi` 错误，全部是 `java.time.LocalDate` / `LocalDateTime` / `DayOfWeek` 等 API 26+ 调用在 minSdk 24 上不兼容。

**选项对比**：
- (A) 把所有 `LocalDate.now()` 替换成 `Calendar` / 三方 backport —— 工作量大、可读性下降
- (B) 抬 minSdk 到 26 —— 损失 ~2% 设备
- (C) 启用 core library desugaring —— 一行配置，全语种一次性解决

选 (C)：`desugar_jdk_libs:2.1.5` + `compileOptions { isCoreLibraryDesugaringEnabled = true }`。Lint 一次跑过。

adaptive icon 模板（108×108dp，安全区中心 72×72）：

旧的 `mipmap-*/ic_launcher_foreground.webp` 5 个文件被替换为 vector drawable 后变孤儿，删除；旧 `drawable/ic_launcher_background.xml`（默认 grid 模板）未被引用，删除。
