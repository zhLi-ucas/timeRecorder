## §1 记一笔界面：滑选式输入整合

### 1.1 诉求

用户原话（2026-06-26）：

> 记一笔界面，为了让一页刚好显示"记一笔"的全部内容，调整如下：
> - 日期栏不再显示"日期"两个字，而是直接显示当前记一笔的日期
> - 一级分类变成滑动选择（左右滑动，有淡入淡出效果，同一时间显示三个，中间为选择项，加深提示，左右为浅色，滑动形式为列表，有头尾，不要循环）

> - 时长也类似地做成滑动操作，默认为 30min

### 1.3 设计方案

#### A. 新增组件 `ChoiceWheel`（横向滑选器）

放在 `ui/components/ChoiceWheel.kt`。从 `WheelPicker` 的思路迁移到横向：

- `LazyRow` + `rememberSnapFlingBehavior(lazyListState)` → snap + fling 物理直接继承
- 3 行可见，正中为选中项；左/右各 1 个淡色 hint
- 渐变 alpha：distance 0 → 1.0（加粗 + 主色），distance 1 → 0.5，distance ≥2 → 0.3（理论上仅 3 个可见，但 LazyRow 渲染窗口外会有 ±1，所以也压暗）
- 顶部/底部各 1 条 `HorizontalDivider`（或左右 `VerticalDivider`）框出选中槽位
- **不循环**：LazyRow 天然有头尾，fling 到边界自然停止
- API 形如：

  ```kotlin
  @Composable
  fun <T> ChoiceWheel(
      items: List<T>,
      selectedIndex: Int,
      onSelectedChange: (Int) -> Unit,
      modifier: Modifier = Modifier,
      itemWidth: Dp = 96.dp,
      label: (T) -> String,
      colorForKey: ((T) -> Any?)? = null,   // 用于一级分类的 colorKey 着色
  )
  ```

  泛型 + `label` lambda 避免每处都做 string mapping。

#### C. 标题派生

`RecordViewModel.save` 时拼装 title：
- L1 + L2 都有：`"{L1}·{L2}"`（默认分隔符中点，待确认）
- 仅 L1（无 L2 子项）：`"{L1}"`
- L2 未选但 L1 有子项：与上同，视为仅 L1

`RecordFormState.title` 字段保留还是删除？
- **方案 A（推荐）**：删除 `title` 字段，DB 写入时直接 `deriveTitle(parent, child)`。编辑老条目时若 DB 中有自定义标题，会被覆盖为派生值——可接受（用户已表态"标题不可自选"）。
- **方案 B**：保留字段，仅 UI 隐藏；编辑时尊重 DB 旧值，新建时派生。复杂度高、收益低。

→ 默认走 A，待用户确认。

#### D. 时长 wheel 的候选集

候选值需要敲定，几个候选方案：
- **方案 1**：5min 步进，0–480min（共 97 项，过长，fling 滑不到底）
- **方案 2**：常用档位 `[15, 30, 45, 60, 90, 120, 150, 180, 240]`（共 9 项，覆盖大部分场景）
- **方案 3**：5min 步进 0–180 + 30min 步进 180–480（混合，34 项）
- 默认值统一为 30min（用户指定）

→ **推荐方案 2**（紧凑、覆盖实际用法）。待用户确认。

#### E. 二级分类 wheel 的"空选项"

L2 是 L1 的子集，切换 L1 时 L2 列表会变。两种处理：
- 切换 L1 → L2 列表重置，selectedIndex 归零（默认选 L1 下第一个 L2）
- 切换 L1 → L2 列表首项加一个"无"选项，selectedIndex 指向"无"

→ 倾向第一种（更省事，且标题派生 L1+L2 自然成立）。待用户确认。

#### F. 备注字段交互

- `singleLine = true` → 内部不再换行
- `imeAction = ImeAction.Done` + `KeyboardActions(onDone = { keyboardController.hide() })` → 回车键直接收起键盘
- `Modifier.fillMaxWidth()`，不再 `minLines = 2`
- 保留 label "备注（可选）"

### 1.6 影响面 / 风险

- **不动 Room schema**：标题派生是写入时的纯逻辑，DB 列保留
- **不动 ViewModel 责任边界**：`RecordViewModel` 仍负责 form state + save；只是字段集合变小
- **WheelPicker 复用**：`DurationPickerSheet` 内的双轮 wheel **保持原样**（它是 5 行纵向，本次不动）；新组件是 `ChoiceWheel`，独立文件
- **Editing 老数据**：DB 中已存在的 `title` 字段在被编辑后会被覆盖；新建场景没问题
- **单屏放下约束**：横向 wheel 占用纵向空间约 `wheel 高度（≈ 80dp）× 3 = 240dp`，加日期按钮 + 备注 + 保存 ≈ 总高 450dp 左右，主流手机视口（≥ 600dp 小屏）放得下；折叠屏 / 小屏需后续验证

### 2.3 设计方案

#### A. 网络栈选型

| 选项 | 优点 | 缺点 | 推荐 |
|---|---|---|---|
| `HttpURLConnection` (JDK) | 零依赖 | API 啰嗦，无内置超时/重试拦截器 | × |
| **OkHttp 4** | 简洁、成熟、mirror 速度快 | 新依赖（~700KB） | ✓ |
| Ktor client | 协程原生 | 依赖体积更大、配置项更多 | × |

→ 用 **OkHttp**。加到 `gradle/libs.versions.toml` + `app/build.gradle.kts`。Aliyun mirror 已配置，下载不是问题。

#### B. API key 与配置存储

存 `app_settings`，三个 key：

| key | 含义 | 默认 |
|---|---|---|
| `deepseek_api_key` | API 密钥（明文） | 空 |
| `deepseek_model` | 模型 ID | `deepseek-chat` |
| `deepseek_enabled` | 是否启用 AI 按钮（用于懒确认隐私提示） | `false` |

**安全说明**：Room DB 内明文存 API key 等同于 root 设备可读；对柳比歇夫自用 App 风险可接受。如需更稳，可后续切到 `EncryptedSharedPreferences`（v1.3 候选，本节不实施）。

#### C. 新模块结构

```
data/
  remote/
    DeepSeekApi.kt          # HTTP + JSON 封装；suspend fun chat(messages, jsonMode): String
    DeepSeekConfig.kt       # data class(apiKey, model)
    AiReviewRequestBuilder.kt  # 把 TimeEntry 列表 + 类别 + 历史复盘拼成 messages
```

ViewModel 层新增 `ReviewAiViewModel`（或挂到 `ReviewViewModel` 上）：

```kotlin
class ReviewViewModel(application, private val settingsDao, ...) {
    private val _aiState = MutableStateFlow<AiState>(AiState.Idle)
    val aiState: StateFlow<AiState>

    fun generateWithAi() {
        viewModelScope.launch {
            _aiState.value = AiState.Loading
            try {
                val cfg = readDeepSeekConfig() ?: run {
                    _aiState.value = AiState.Error("未配置 API key")
                    return@launch
                }
                val snapshot = buildPeriodSnapshot()       // 当前周期的条目 + 历史
                val messages = AiReviewRequestBuilder.build(snapshot, _periodType.value)
                val raw = DeepSeekApi(cfg).chat(messages, jsonMode = true)
                val parsed = parseAiReviewJson(raw)        // {summary, findings, adjust}
                _form.value = _form.value.copy(
                    summaryText = parsed.summary,
                    mainFindings = parsed.findings,
                    adjustmentPlan = parsed.adjust,
                )
                _aiState.value = AiState.Done
            } catch (e: Exception) {
                _aiState.value = AiState.Error(e.message ?: "未知错误")
            }
        }
    }
}

sealed interface AiState {
    data object Idle : AiState
    data object Loading : AiState
    data object Done : AiState
    data class Error(val msg: String) : AiState
}
```

fun ColorSwatchPicker(

@Composable
private fun ColorSwatch(color: Color, selected: Boolean, onClick: () -> Unit) {

            .size(40.dp)
            .clip(CircleShape)
            .background(color)
            .clickable(onClick = onClick)
            .then(
